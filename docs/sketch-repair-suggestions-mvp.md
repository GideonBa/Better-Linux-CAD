# Sketch repair suggestion seed

Status: implemented seed. This is not a sketch solver and does not mutate model intent.

This document describes the first non-mutating sketch repair suggestion layer. It consumes `SketchDiagnosticReport` records and emits deterministic candidate actions that a future UI or CLI can show to the user.

## Goal

The goal is to bridge the first sketch diagnostics layer and a future explicit repair/apply command system.

The repair-suggestion layer should:

- read diagnostics generated by `SketchConstraintDiagnostics`
- produce stable `SketchRepairSuggestion` records
- link every suggestion back to the originating diagnostic kind and target
- serialize suggestion reports as debug/output JSON
- avoid changing sketches automatically
- avoid storing suggestions in `.blcad.json` model intent

## Implemented records

The implemented API lives in `include/blcad/core/sketch_repair_suggestions.hpp`.

```text
SketchRepairSuggestionAction
  AddFixedEndpointConstraint
  RemoveConflictingOrientationConstraint
  RemoveDuplicateFixedEndpointConstraint
  RemoveDuplicateDrivingDimension
  AddDrivingDimension

SketchRepairSuggestion
  action
  target
  originating_diagnostic_kind
  originating_diagnostic_target
  message

SketchRepairSuggestionReport
  sketch_id
  suggestions[]

SketchRepairSuggester
  suggest(SketchDiagnosticReport) -> SketchRepairSuggestionReport
```

## Mapping from diagnostics to suggestions

The first deterministic mapping is intentionally small:

```text
UnconstrainedLineEndpoint
  -> AddFixedEndpointConstraint

ConflictingHorizontalVerticalConstraints
  -> RemoveConflictingOrientationConstraint

DuplicateFixedEndpoint
  -> RemoveDuplicateFixedEndpointConstraint

DuplicateDrivingDimensionTarget
  -> RemoveDuplicateDrivingDimension

ProfileWithoutDrivingDimensions
  -> AddDrivingDimension
```

The seed deliberately does not generate repairs for `FreeSplineControlPoint` or `RedundantOrientationConstraint` yet. Those require more context to avoid poor automatic advice.

## Debug JSON output

Suggestion reports can be serialized with:

```text
serialize_sketch_repair_suggestion_report_to_json(report)
```

The output schema is:

```text
blcad.sketch_repair_suggestions.debug
```

Example shape:

```json
{
  "schema": "blcad.sketch_repair_suggestions.debug",
  "version": 1,
  "sketch": "sketch.main",
  "suggestion_count": 1,
  "suggestions": [
    {
      "action": "add_fixed_endpoint_constraint",
      "target": "line.a:start",
      "originating_diagnostic_kind": "unconstrained_line_endpoint",
      "originating_diagnostic_target": "line.a:start",
      "message": "add an explicit fixed endpoint constraint for this currently unconstrained endpoint",
      "mutates_model": false
    }
  ]
}
```

This JSON is an output/report artifact only. It must not be persisted as part of normal `.blcad.json` model intent.

## Test coverage

The seed has core tests for:

- fixed-endpoint suggestions for unconstrained line endpoints
- orientation-conflict removal suggestions
- duplicate fixed-endpoint removal suggestions
- duplicate driving-dimension removal suggestions
- add-driving-dimension suggestions for profile sketches without dimensions
- debug JSON serialization
- deterministic links to originating diagnostic kinds and targets

## Deliberate limitations

This block does not implement automatic model repair.

It does not add or remove constraints in a `Sketch`.

It does not create parameter objects for suggested dimensions.

It does not decide which side of an orientation conflict should be removed.

It does not solve degrees of freedom, generate exact solver plans, repair spline handles, or rewrite `.blcad.json` documents.

A later block should add explicit apply commands that can execute a selected suggestion only after the caller deliberately chooses it.
